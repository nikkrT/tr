# E-Commerce Microservices API

Пет-проект, демонстрирующий построение масштабируемой микросервисной архитектуры на языке **Go**. Проект представляет собой бэкенд (API) для организации, которая работает с заказами и товарами.

## 🏗 Архитектура

Проект разбит на несколько независимых микросервисов, каждый из которых отвечает за свою бизнес-область. Сервисы общаются между собой синхронно через **gRPC** и асинхронно через брокер сообщений **RabbitMQ**. Каждое приложение построено с использованием принципов **Clean Architecture**.

### Компоненты системы:

1. **API Gateway (`gateway`)**
    - Единая точка входа для всех клиентских HTTP-запросов.
    - Проксирует запросы и транслирует REST/HTTP-вызовы во внутренние gRPC-запросы к микросервисам.
    - Навайбкожен
2. **Product Service (`productService`)**
    - Управление каталогом товаров (CRUD операции).
    - Предоставляет gRPC API для других сервисов (например, для проверки наличия товара).
    - Публикует события об изменении товаров в RabbitMQ.
3. **Order Service (`orderService`)**
    - Управление заказами.
    - При создании заказа синхронно обращается к `productService` по gRPC для проверки доступности товара.
4. **Notification Service (`notificationService`)**
    - Сервис уведомлений.
    - Асинхронно слушает очереди в RabbitMQ и реагирует на события (например, создание заказа или добавление товара).

## 🚀 Стек технологий

- **Язык:** Go (Golang)
- **Транспорт & API:** REST (go-chi/chi), gRPC, Protocol Buffers
- **Брокер сообщений:** RabbitMQ
- **База данных:** PostgreSQL (с использованием `pgxpool`)
- **Миграции:** golang-migrate
- **Инфраструктура:** Docker, Docker Compose

## 📁 Структура проекта

```text
├── gateaway/             # API Gateway сервис
├── productService/       # Микросервис товаров
├── orderService/         # Микросервис заказов
├── notificationService/  # Микросервис уведомлений
├── pkg/                  # Общие пакеты (модели, proto-файлы сгенерированные для gRPC)
├── init-scripts/         # Скрипты инициализации баз данных
└── docker-compose.yml    # Описание инфраструктуры
```

Внутри микросервисов применяется слоистая архитектура:
- `application/` — инициализация приложения, DI, серверы (HTTP, gRPC).
- `business/` (или `service/`) — бизнес-логика и юзкейсы.
- `infrastructure/` — работа с БД (Postgres, репозитории) и брокерами (RabbitMQ).
- `interfaces/` — хендлеры REST API и gRPC серверы.
- `config/` — управление конфигурацией.

## 🛠 Как запустить локально

Требования: установленные **Docker** и **Docker Compose**.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/nikkrT/tr.git
   cd tr
   ```

2. Запустите всю инфраструктуру и микросервисы одной командой:
   ```bash
   docker-compose up --build -d
   ```

3. Docker Compose автоматически поднимет:
    - PostgreSQL (на порту `5432`)
    - RabbitMQ (на портах `5672` и `15672` для UI)
    - Контейнер с миграциями (накатит схемы БД)
    - Все микросервисы и API Gateway (доступен на порту `8084`)

## 📡 Примеры запросов (API)

Все запросы отправляются на API Gateway (`http://localhost:8084`).

**Создание заказа:**
```bash
curl -X POST http://localhost:8084/api/v1/orders \
  -H "Content-Type: application/json" \
  -d '{"product_id": 1, "quantity": 2}'
```

*(Добавьте сюда другие примеры запросов, например, для получения списка товаров, как только они будут реализованы в Gateway).*
